Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • If there are 3 elements nested inside a body element, these 3 nested elements are also called child elements.
  • The element that holds the child elements are also known as the parent element.
  • The parent and child language is used often by developers to explain the relationship among elements in HTML.
  • The section element defines a section in a document that contains thematic content.
  • The image element contains information about images that are displayed on the webpage.
  • The header element appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • The paragraph element represents a paragraph or block of text.
  • The unordered lists, ordered lists, and list items represent bulleted or numbered lists.
  • The src attriibute defines the location of the image.
  • The line break element creates a line of empty space, or a line break between two blocks of content.
  • The anchor attribute creates links to the same webpage or other webpages.
  • The alt attribute contains a text string that describes the appearance and fuctionality of the image.
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Text align is one way to center a title or heading on a webpage.
  • The img selector assigns property values to any image element in the markup.
  • The display property and block value assigns each image to start on a new line.
  • box shadow adds shadow effects around an elements frame.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch name: creates a new branch and switches to it.
  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Git commands always start with git and are used to save work and track versions of your code.
